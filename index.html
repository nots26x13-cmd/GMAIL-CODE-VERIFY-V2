Overview

Two endpoints. One flow.

Everything this API does maps to a single, ordered flow — an email comes in, a code goes out, and comes back for a check. That order is the whole design.

1. Email inPOST /api/send-code
2. Gmail sends codeApps Script backend
3. Code enteredPOST /api/verify-code
4. Access grantedYour app decides
Endpoints

Reference

Both endpoints accept GET or POST. Use POST with a JSON body in production; GET with query params is there for quick manual testing.

POST /api/send-code

Generates and emails a verification code to the given address.

Parameters
FieldTypeRequired
emailstringyes
subjectstringno — defaults to "Your verification code"
messagestringno
Request
curl -X POST https://gmail-code-verify-v2.vercel.app/api/send-code \
  -H "Content-Type: application/json" \
  -d '{"email":"user@example.com"}'
Response
{
  "success": true
}
POST /api/verify-code

Checks whether a code matches what was sent to that email.

Parameters
FieldTypeRequired
emailstringyes
codestringyes
Request
curl -X POST https://gmail-code-verify-v2.vercel.app/api/verify-code \
  -H "Content-Type: application/json" \
  -d '{"email":"user@example.com","code":"123456"}'
Response
{
  "success": false,
  "error": "Code expired"
}
Status codes
CodeMeaning
